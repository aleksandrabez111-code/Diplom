# Объяснение упрощённой версии проекта

Проект специально упрощён под диплом:

1. Backend API на FastAPI.
2. Виджет на чистом HTML+JavaScript.
3. Простая внутренняя страница на чистом HTML+JavaScript.

Без React, без сложной сборки фронтенда, без журнала действий и лишних функций.

---

## Для чего нужна каждая часть

## 1) `backend`

Это серверная часть, где хранится вся логика:
- принимает обращения из виджета;
- хранит заявки и темы в SQLite;
- выполняет вход сотрудников по JWT;
- отдаёт список заявок;
- позволяет взять заявку в работу и закрыть её.

Проще говоря: backend — это «мозг» и «база памяти» проекта.

## 2) `simple-frontend/embed-widget-snippet.html`

Это готовый код виджета для вставки в любой HTML-шаблон сайта.

Зачем он нужен:
- пользователь без регистрации выбирает тему;
- пишет текст обращения;
- оставляет email;
- отправляет заявку.

Плюс: это обычный HTML+JS блок, его можно просто скопировать и вставить в страницу.

## 3) `simple-frontend/simple-panel.html`

Это максимально простая внутренняя панель для сотрудников.

Что умеет:
- логин;
- просмотр заявок;
- фильтр по статусу;
- кнопки «взять», «в работу», «закрыть».

То есть это минимальный рабочий интерфейс без сложной админки.

---

## Почему это удобно для диплома

- минимум технологий на фронтенде (только HTML/JS);
- код легко объяснить на защите;
- меньше риск поломок на демонстрации;
- всё запускается быстро локально.

---

## Что было убрано при упрощении

Убраны, чтобы не перегружать диплом:
- отдельная React-админка;
- отдельная сборка виджета как отдельный frontend-сервис;
- журнал действий;
- архив;
- передача заявок между специалистами;
- SMTP-рассылки.

Осталось только то, что нужно для демонстрации базового процесса обработки обращений.

---

## Как теперь работает процесс

1. Пользователь открывает виджет на сайте.
2. Отправляет обращение.
3. Backend сохраняет заявку в БД и даёт номер.
4. Сотрудник входит в `simple-panel.html`.
5. Видит заявку, берёт в работу, меняет статус.
6. После решения переводит в `closed`.

---

## Что где находится

- API сервер: `backend/app/main.py`
- Публичные маршруты: `backend/app/api/public.py`
- Заявки: `backend/app/api/tickets.py`
- Пользователи: `backend/app/api/users.py`
- Темы: `backend/app/api/topics.py`
- Миграции БД: `backend/alembic/versions/`
- Seed-данные: `backend/scripts/seed.py`
- Встраиваемый виджет: `simple-frontend/embed-widget-snippet.html`
- Простая панель: `simple-frontend/simple-panel.html`

## Итог

Получился небольшой, понятный и демонстрационный проект:
- backend с базовой бизнес-логикой;
- фронтенд без сборки и сложных зависимостей;
- достаточный функционал для дипломной презентации.
